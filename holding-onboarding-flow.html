CRS Entity Classification Definitions

ClassificationDefinitionExamples
Financial Institution (FI) Bank, investment fund, insurance company, custodial institution Currently out of VMSA risk appetite
Active NFE ≥50% of gross income from active sources (sales, services) OR ≥50% of assets used in active business Trading companies, SaaS, retailers, holdings managing active subsidiaries
Passive NFE Does not qualify as Active and is not an FI. Income mainly from dividends, interest, royalties, rental income Dormant holdings, IP entities, SPVs, family offices
Key nuance
Income from actively managed operating businesses = active income. Shareholdings in active subsidiaries = active assets (not passive).
Key Design Decisions
Rationale behind each design choice in the proposed flow.

NACE code as trigger, not sole decider

NACE codes can be wrong — set at company registration and rarely updated. A consulting firm might have a holding NACE code, or a real holding might be registered as "management consultancy." We use NACE 6420/7010 to trigger the question, but the client confirms whether they are actually a holding.

Inline questions replace async inquiry

Current: application → acceptance request → CORPORATE_STRUCTURE inquiry → client replies by email → agent review → decision.
Proposed: questions answered in onboarding flow → immediate automated decision. This eliminates the compliance ops bottleneck for holdings entirely.

Subsidiary capture via CreditSafe search

Client searches companies by name in a search box (powered by CreditSafe API), selects them, and enters ownership %. This data is stored internally and populates the entity graph with Company → OWNS → Company edges with ownership percentages and source tracking.

Passive NFE = onboard + flag (policy change)

Previously: passive holdings were rejected.
Now: passive NFEs can be onboarded but must be flagged internally. The Passive NFE flag is back-office only (not visible to client) and must be updateable if classification changes over time (e.g., company acquires active subsidiaries).

Active/Passive test = CRS standard (OECD)

Uses the OECD CRS definition: ≥50% income from active sources OR ≥50% assets in active business = Active NFE. This is an international regulatory standard, not an internal invention. It directly satisfies CRS reporting obligations while providing the substance test we need.

"Plans to own" branch

A company with a holding NACE code that confirms it's a holding but doesn't currently own subsidiaries — if they plan to acquire, we onboard them as Passive NFE. This prevents rejecting companies in early formation stages that intend to build a corporate group.

CRS data: optional now, mandatory later

All new CRS fields (tax residence, TIN, place of birth, entity classification) are added as optional during initial rollout. This prepares the system for when CRS reporting becomes mandatory without blocking current onboarding. Applies to new clients only.
